## Change Tracking
- Keep track of all changes and updates to the project.
- Maintain a comprehensive log of modifications in `CHANGELOG.md`.

## Changelog
- Maintain a `CHANGELOG.md` file following the [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) convention.
- Log every meaningful change under `Unreleased` section during development.
- Include the reasoning behind changes (why, not just what).

## Dependency Tracking
- Keep track of all dependencies and their versions.
- Regularly check for updates and security vulnerabilities.
- Document any manual dependency updates in `CHANGELOG.md`.

## Log View
- Build a log view for all changes and prompts to the model.
- Ensure that the reasoning behind changes is documented (e.g., in commit messages or adjacent documentation).
- Maintain a history of architectural decisions in `ARCHITECTURE.md` or similar.
